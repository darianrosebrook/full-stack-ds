# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## What this repo is

This is **not** primarily a design system — it is a falsifiable architectural claim about contract-governed compositional systems, with a Web DOM design system as the existence proof. One polymorphic primitive (`Stack`) plus JSON contracts drive code generation for **five** framework targets (React, Vue, Svelte, Angular, Lit) plus a Figma descriptor target. If the same contract can drive all five idiomatically without leaking framework details, the contract is at the right level of abstraction.

Read `docs/current-implementation-snapshot.md` first to know what is currently proven vs. only foundation. That document overrides older architecture docs when they disagree.

## Commands

Package manager is **pnpm@10.14.0** (enforced). Node 22 in CI. `npm`/`yarn` lockfiles are gitignored.

```bash
# Showcase app (React)
pnpm run dev
pnpm run build

# Codegen (default target = react)
pnpm run generate                              # React only
pnpm run generate -- --target=all              # All five frameworks + Figma descriptors
pnpm run generate -- --target=vue,svelte       # Subset
pnpm run generate -- Switch                    # Single component
pnpm run generate -- --target=all Switch Dialog
pnpm run generate:watch
pnpm run generate:dry-run                      # Preview paths, write nothing

# Validation — escalating strictness
pnpm run generate:validate                     # Schema shape only
pnpm run generate:check                        # Schema + cross-graph semantic checks (THE real gate)
pnpm run generate:validate-usage               # + usage JSONL refs

# Governed admission rail (proves generated bytes match contract+codegen+env)
pnpm run governed:rail                         # Regenerate-all then verify in required mode
pnpm run governed:rail:changed                 # Above + git-scoped projection for reviewers

# Tests — root is Vitest; each framework package has its own
pnpm test                                      # Root vitest run
pnpm run test:vue                              # Vue package
pnpm run test:svelte
pnpm run test:angular
pnpm run test:lit
pnpm run test:frameworks                       # vue → lit → svelte → angular (sequential)
pnpm run test:all                              # root + all framework packages

# Single-test invocation
pnpm exec vitest run path/to/file.test.ts
pnpm exec vitest run -t "test name pattern"
pnpm --filter @full-stack-ds/vue exec vitest run path/to/file.test.ts

# Type checking (separate per target because tsconfig differs)
pnpm run typecheck                             # root + codegen
pnpm run typecheck:vue
pnpm run typecheck:svelte
pnpm run typecheck:angular
pnpm run typecheck:lit
pnpm run typecheck:all

# Linting
pnpm run lint
pnpm run lint:fix

# Tokens (DTCG source → composed → resolved)
pnpm run tokens:build                          # Emits composed.tokens.json (gitignored) + tokens.css
pnpm run tokens:validate                       # DTCG validation of source
pnpm run tokens:build:check                    # Build + diff-exit-code on committed output
pnpm run tokens:check-contrast                 # WCAG AA gate on curated pairs
pnpm run tokens:check-brand-refs               # All {ref}s in brand overrides resolve
pnpm run tokens:check-usage:gate               # Regression gate vs. usage-baseline.json
pnpm run tokens:write-usage-baseline           # Re-baseline (commit the result)

# Playwright runtime fact rail (React/Vue/Svelte/Lit preview mounts)
pnpm run e2e:rail
pnpm run e2e                                   # Full Playwright suite
```

### Pre-push hook mirrors CI

`.githooks/pre-push` runs the same 13-step sequence as `.github/workflows/ci.yml`. Activate per-clone with `git config core.hooksPath .githooks` (the `prepare` script attempts this automatically). If the local hook passes, CI should pass. Do not `--no-verify` to dodge gate failures — fix the underlying issue.

### CAWS runtime hooks govern every session (`.Codex/hooks/`)

Separate from the git pre-push hook above, this repo installs the **CAWS Codex hook pack** under `.Codex/hooks/`. These fire on `PreToolUse` / `PostToolUse` / `SessionStart` / `Stop` / `PreCompact` (wired in `.Codex/settings.json` → `caws_dispatch/`) and actively gate your work — some **block** tool calls. The ones most likely to stop you:

- **block-dangerous.sh** (`Bash`) — denies catastrophic commands and arms a sticky per-session **danger latch** on `ask:confirm`; once latched, all mutating commands block until the user runs `reset-danger-latch.sh`.
- **scope-guard.sh** (`Write`/`Edit`) — out-of-scope edits escalate by **strike** (advise → ask → block); fix with `caws specs amend-scope`, clear stale strikes with `reset-strikes.sh`.
- **worktree-guard.sh** / **worktree-write-guard.sh** / **bash-write-guard.sh** — block history-rewriting git and cross-worktree / claimed-path writes via the shared claim oracle.
- **protected-paths.sh** — hard-blocks `Write`/`Edit` under `.Codex/hooks/*` and to strike-state files.

**Full reference (every active hook, the dormant ones, escape hatches, state-file locations): [`.Codex/hooks/README.md`](.Codex/hooks/README.md).** Hook scripts are CAWS-managed (`do_not_edit_directly`) — change them via `caws init`, never by editing in place, and never weaken a guard to get past it.

## Architecture

### Layer authority (read `docs/codegen-authority.md` before touching emitters)

```
Contract              declares component semantics
Semantic IR           normalizes contract into framework-neutral facts
Framework emitter     lowers IR into idiomatic framework syntax
Admission rail        validates emitted artifacts pass framework tooling
Behavioral tests      validate runtime behavior satisfies the contract
```

The central architectural invariant: **a component's durable semantics live in the contract, with each framework as a realization backend.** Emitters must not accumulate hand-authored per-component lore (`if (component === "ShowMore") …`). If you find yourself special-casing a component name in an emitter, the IR is missing a fact — push the fact down into the IR, not up into the emitter.

- Contract knows: "this part is a trigger," "this channel is `open`," "role `option`."
- IR knows: "text-node child binding," "ARIA Booleanish attribute," "host has implicit role."
- Emitter knows: "in Svelte, text children are `{expr}` not `textContent={expr}`."
- Rail knows: "the Svelte compiler rejects this." It does **not** decide "therefore X is special."

### Source-of-truth corpus

Component contracts live at `packages/ds-contracts/components/<Name>/<Name>.contract.json` with optional sidecars:

- `<Name>.tokens.json` — token bindings
- `<Name>.styles.json` — style declarations
- `<Name>.usage.jsonl` — curated documentation examples (rendered in the Design view; **not** codegen input)

The filesystem loader (`packages/ds-codegen/src/contracts-fs.ts`) is authoritative for corpus discovery — never trust hand-written component counts in prose/docs.

The single primitive contract: `packages/ds-contracts/primitives/Stack.primitive.json`. Per-target import paths are declared in `implementation.targets.<framework>.relativeFromComponents`.

### Monorepo layout

```

  ds-codegen/           # TypeScript CLI + IR + emitters + admission rail
```

### Behavior primitives are mirrored across frameworks

Each framework package ships the same primitive set under different names (`useFocusTrap` / `createFocusTrap` / `FocusTrapController`). The API contract is identical — same options shape, same channel keys — so codegen dispatches from IR to target-specific implementations. If you add a behavior primitive, add it to all five.

### Generated artifact admission rail

The rail binds emitted bytes to four evidence rungs: artifact, contract, codegen-source, environment. `pnpm run governed:rail` regenerates everything and then verifies in required mode; the verifier emits typed `RAIL_*` diagnostic codes when integrity claims drift. CI also runs `git diff --exit-code` over the five generated `src/` trees after the rail — proves the regenerated bytes were already committed.

The emission manifest at `packages/ds-codegen/.emission-manifest.json` is **gitignored** runtime state. It is "what the last `generate` run produced on this machine." Schema migrations are regenerate-driven (no data migration) — a stale manifest triggers `RAIL_REQUIRE_MANIFEST_SCHEMA_MISMATCH`, repair is `pnpm run generate -- --target=all`.

Read `docs/specifications/admission-rail.md`, `docs/specifications/governed-ci.md`, `docs/specifications/manifest-schema.md` before touching anything under `packages/ds-codegen/src/validation/`.

### Token graph

`composed.tokens.json` and `resolved.tokens.json` are gitignored regenerable scratch — every CI/pre-push run starts with `pnpm -F @full-stack-ds/tokens build` before running `generate:check`, because the semantic check loads the composed graph to verify contract `resolvesTo` paths. Without that build, every contract reports `token graph not built` DRIFT.

Brand override files do **not** participate in the main token build's ref validation (they layer separately). `tokens:check-brand-refs` is the gate that catches unresolved `{ref}`s in brand overrides — without it, a typo like `{color.palette.cobalt.500}` silently no-ops at runtime.

## Common editing workflows

### Editing an existing component

1. Edit `packages/ds-contracts/components/<Name>/<Name>.contract.json` and any sidecars.
2. `pnpm run generate:check` (and `generate:validate-usage` if you changed usage).
3. `pnpm run generate -- --target=all <Name>`.
4. `pnpm test && pnpm run test:frameworks && pnpm run governed:rail`.

**Do not hand-edit generated component files** unless the edit is inside an admitted `@custom:start` / `@custom:end` block (see `packages/ds-codegen/src/preserve.ts`). CSS and pure scaffolding are regenerated and will be overwritten. `--force` exists as an emergency override; using it is a signal that the contract should have carried the change instead.

### Adding a new component

1. Create `packages/ds-contracts/components/<Name>/<Name>.contract.json` against `component.contract.schema.json`.
2. Add sidecars only when needed.
3. `pnpm run generate:check` and `pnpm run generate -- --target=all <Name>`.
4. Add to showcase navigation only if it should appear in the app.

### Schema / IR / emitter changes

If you add or rename a contract field:
1. Update the schema under `packages/ds-contracts/`.
2. Update `packages/ds-codegen/src/ir.ts` so the field has a governed IR representation.
3. Update each affected emitter under `packages/ds-codegen/src/frameworks/<target>/`.
4. Regenerate all and run `pnpm run governed:rail`.

Emitters must not re-parse raw contract fields when the IR should own the translation.

### Adding a new framework target

Built-in targets are admitted via `packages/ds-codegen/src/target-packs/builtin.ts` and need a per-framework admission plan in `packages/ds-codegen/src/validation/frameworks/<target>.ts`. Local target packs declared in `fsds.targets.json` are currently **metadata-only** — their emitter entrypoints are not loaded/executed yet. See `docs/architecture/design/target-pack-registry.md`.

## Non-claims to respect

When reasoning or writing docs/comments, do not over-claim:

- The rail proves **artifact↔contract↔codegen↔env binding**, not determinism, not visual quality, not a11y adequacy, not full environment attestation.
- The runtime fact rail (Playwright) proves narrow defaults for Progress/Truncate/ShowMore/OTP/Calendar across React/Vue/Svelte/Lit. **Angular runtime preview is skipped.** Screenshot baselines are darwin-only and skip under `CI=true` — CI runs OS-agnostic fact assertions.
- Non-web targets (Swift, Jetpack Compose, React Native) are recon/prototype. Web DOM is the only admitted executable family.
- `@full-stack-ds/*` packages are **workspace-only**; not published to npm.
- Local target packs are metadata-only until an executable local-loader slice lands.
- Figma descriptor emission ≠ live Figma library publication.

## Conventions specific to this repo

- **No shadow files** — no `*-enhanced.*`, `*-v2.*`, `*-new.*`. Edit in place.
- **No `package-lock.json` or `yarn.lock`** — gitignored to keep pnpm authoritative.
- **`pnpm run prepare`** wires `core.hooksPath = .githooks` so the pre-push gate activates on first install.
- **Conventional commits** with scopes that name the area: `feat(figma-plugin): …`, `fix(e2e): …`, `docs(architecture): …`, `wip(<scope>): checkpoint …`.
- Tests that produce visual evidence (screenshots) write under `test-results/` (gitignored) and `tmp/` (gitignored, project scratch).
- When a script touches `composed.tokens.json` or anything under `packages/ds-codegen/.emission-manifest.json`, it is touching gitignored regenerable state — never commit those files.
