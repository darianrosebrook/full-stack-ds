# @full-stack-ds/figma-plugin

Figma desktop plugin for reviewing and materializing Full Stack DS component contracts into Figma component sets, placeholder components, and descriptor-backed metadata. It also registers a Dev Mode codegen handler that renders per-framework code previews for a materialized selection.

The plugin consumes descriptors generated by `@full-stack-ds/codegen --target=figma` under `src/generated/components`. Those descriptors are transfer artifacts derived from `ComponentIR`; they are not source of truth.

## Current scope

- Opens a Svelte-based plugin UI (`src/FigmaPluginApp.svelte`) built with Full Stack DS Svelte components, showing descriptor metadata, materialization status, variant axes, draft variant labels, and audit findings for all generated component descriptors.
- The UI is display-and-command only: it renders `src/ui-model.ts`'s `FigmaUiModel` and posts typed `UiToPluginMessage` commands (`fsds:materialize`, `fsds:resize`, `fsds:close`); it never mutates the Figma document directly.
- Every materialization command runs plugin-side through the plan-driven core: `materializeComponentSet` / `materializeLeafComponent` (`src/plugin.ts`) plus `materializeComponentStateSurface` (`src/materialize-state.ts`, itself driven by `planFigmaStateSurface` in `src/planner.ts`).
- Component-set eligibility (`classifyDescriptorForMaterialization` + `COMPONENT_SET_ALLOWLIST`) is decided by the single shared module `src/eligibility.ts`, consumed by both `plugin.ts` and `ui-model.ts`.
- A headless run path (`main()` with default options) still completes materialization and closes the plugin without any UI round-trip — used by tests and scripted invocations.
- In Figma Dev Mode (`figma.editorType === "dev" && figma.mode === "codegen"`), registers `figma.codegen.on("generate", ...)` (`src/plugin.ts`'s `setupCodegen`) which resolves the descriptor + variant selection from `fsds.component` / `fsds.variantRow` plugin data and renders a code preview per target framework via `src/codegen-preview.ts`.
- Creates the Stack primitive component set used by placeholder anatomy instances.
- Keeps description and variant-label edits as UI-local drafts; there is no contract-writeback path yet.

## Build

```bash
pnpm --filter @full-stack-ds/figma-plugin run build
```

The build runs two Vite passes (`vite build --mode ui && vite build --mode plugin`, see `vite.config.ts`):

1. **`ui` mode** builds the Svelte UI (`src/ui.html` entry) to `dist/ui.html` + `dist/assets/*`, then an `inlineUiAssets` plugin inlines the built JS/CSS directly into `dist/ui.html` and deletes `dist/assets` — the Figma UI iframe cannot fetch external asset files at runtime, so the HTML must be fully self-contained.
2. **`plugin` mode** builds `src/plugin.ts` to `dist/plugin.js` (an IIFE bundle), baking the now-inlined `dist/ui.html` into a `__html__` string constant (`figma.showUI(__html__)`) via Vite's `define`. If `dist/ui.html` doesn't exist yet when this pass runs, `readUiHtmlForPlugin()` falls back to `<main>Full Stack DS plugin UI failed to load.</main>` — the plugin still opens rather than crashing.

Build output is `dist/plugin.js` plus a self-contained `dist/ui.html`. Load `manifest.json` from this package in Figma Desktop during development.

> [!IMPORTANT]
> **Rebuild before you load.** `dist/plugin.js` and `dist/ui.html` are gitignored build output and
> Figma loads them directly from disk — nothing rebuilds them for you. If you edit
> `src/**` (or regenerate the descriptors under `src/generated/`) and don't
> rebuild, Figma runs the **old** bytes. This is how the
> `combineAsVariants: Grouped nodes must be in the same page as the parent`
> error came back after the fix had already landed in source.
>
> - Active development: `pnpm --filter @full-stack-ds/figma-plugin dev`
>   (builds the UI once, then Vite `--watch` on the plugin bundle).
> - One-off / before loading: run `build`, then verify freshness with
>   `pnpm --filter @full-stack-ds/figma-plugin build:check`
>   (fails if any `src/**` input is newer than `dist/plugin.js`).

## Why some UI imports are relative, not package-form

`src/ui.ts` and `src/FigmaPluginApp.svelte` import Full Stack DS Svelte components via relative paths (`../../ds-svelte/src/components/...`) rather than `@full-stack-ds/svelte`. That package's `exports` map only resolves to `dist/*` (built via `svelte-package`), and that `dist/` is not committed and is out of this package's build to produce. `vite.config.ts`'s `ui` mode config aliases the package specifier to `../ds-svelte/src/index.ts` for JS/TS imports, but plain CSS imports bypass that alias, so the relative paths are used directly for both.
