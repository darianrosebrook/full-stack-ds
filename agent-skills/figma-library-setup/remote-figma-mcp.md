# Remote Figma MCP Skill Surface

Use this surface for cloud/file inspection and remote metadata. It is read-first by default.

## Purpose

The remote Figma MCP is useful for answering:

- Which file is the target library file?
- What pages, components, component sets, styles, variables, and libraries already exist?
- What naming conventions or collisions must the local materializer respect?
- Is the file already published as a library, and what is its observable metadata?

## Allowed reads

When tools exist, prefer reads in this order:

1. Resolve file identity from URL or file key.
2. Read file metadata.
3. Read page list and top-level node inventory.
4. Read existing component/component-set names.
5. Read style and variable collections.
6. Read comments or review notes only when relevant to setup decisions.

## Writes

Do not perform remote writes by default. Remote Figma MCP writes are allowed only when all are true:

- The user explicitly requested the remote write.
- The tool supports that write directly.
- The change is not better performed in the local desktop file.
- The agent has produced a short change plan naming the exact remote objects affected.

Publishing a library, changing permissions, moving files, deleting files, or editing team-level settings always requires explicit user authorization.

## Evidence to capture

For every remote read, record:

- File key or URL.
- File name.
- Last modified timestamp when available.
- Pages inspected.
- Existing component or component-set count when available.
- Collision candidates.
- Missing tool capabilities.

## Collision handling

If remote state already contains names matching generated descriptors:

- Prefer stable plugin data keys when available from local state.
- Otherwise report collisions before mutation.
- Do not infer that a same-name component is generated by Full Stack DS unless provenance data exists.

## Remote/local consistency check

When both remote and local MCPs are available:

- Compare file key, file name, and page inventory when exposed.
- If they conflict, pause before local mutation.
- If one surface lacks identity data, state the weaker evidence rather than claiming a match.
